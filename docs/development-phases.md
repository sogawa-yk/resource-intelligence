# 開発フェーズ定義書

**プロジェクト**: Resource Intelligence Agent
**作成日**: 2026-03-24
**ブランチ**: `001-oci-resource-intelligence`

---

## フェーズ概要

| フェーズ | 名称 | 目的 | ストレージ |
|----------|------|------|------------|
| MVP | 基本機能 + Chainlit UI | リソース検索・依存関係マップの基本動作をChainlit UIで確認可能にする | ローカルJSONファイル |
| Phase 2 | 作成者特定 + A2A連携 | Audit Log連携による作成者特定、他エージェントとのA2A連携を追加 | ローカルJSONファイル |
| Phase 3 | 本番運用基盤 | Autonomous Database移行、13ヶ月履歴保持、本番可観測性 | Autonomous Database |

---

## MVP（今回のスコープ）

### 含まれる機能

| ID | 機能 | 対応FR | 対応ユーザーストーリー | 説明 |
|----|------|--------|------------------------|------|
| MVP-01 | リソース検索（単一リージョン） | FR-001, FR-002, FR-003 | US1 | 指定リージョン・コンパートメントでのリソース検索。フィルタリング対応 |
| MVP-02 | リソース検索（マルチリージョン） | FR-001, FR-004 | US1 | 全サブスクライブ済みリージョン横断検索。ページネーション対応 |
| MVP-03 | 依存関係マップ生成 | FR-005, FR-006, FR-007 | US2 | 10種類の依存関係タイプの解析。探索深度制御（デフォルト3、最大10） |
| MVP-04 | クロスバウンダリ依存関係 | FR-005 | US2 | コンパートメント・リージョン境界を超えた参照ノード（OCID・名前・タイプのみ） |
| MVP-05 | Chainlit UIフロントエンド | - | 全体 | 自然言語でリソース検索・依存関係マップ生成を指示できるチャットUI |
| MVP-06 | 読み取り専用保証 | FR-011 | 全体 | 全API呼び出しがRead Only操作のみであることの保証 |
| MVP-07 | 基本エラーハンドリング | FR-013, FR-014, FR-015 | US5 | レート制限リトライ、認証エラー処理、部分結果返却 |
| MVP-08 | 構造化ログ | FR-017（部分） | 全体 | 構造化ログ出力（Constitution IV準拠）。W3C traceparentは将来対応 |
| MVP-09 | 履歴保持（ローカルJSON） | FR-016（部分） | 全体 | 検索結果・マップをJSONファイルとしてローカル保存。同日重複は最新のみ |
| MVP-10 | OKEデプロイ | - | 全体 | Dockerfile、Kubernetesマニフェスト、OCIRイメージプッシュ |

### MVPの判定基準

- Chainlit UIからリソース検索を実行し、結果が表示される
- Chainlit UIから依存関係マップ生成を実行し、ノード・エッジが表示される
- 全サブスクライブ済みリージョンを横断した検索が動作する
- OKE上にデプロイして動作する
